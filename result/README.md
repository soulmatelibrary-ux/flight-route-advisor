# result/ — 개발 완료 검증 (초기 문서 ↔ 구현 대조)

- 문서 버전: 1.0
- 작성일: 2026-07-21
- 관련: [../CLAUDE.md](../CLAUDE.md) §8, [../docs/07-checklist.md](../docs/07-checklist.md)

이 폴더는 **개발 완료 시점에 초기 설계 문서(`docs/`) 대비 실제 구현이 일치하는지, 기능이 모두 구현됐는지 검증한 리뷰를 보관**하는 곳이다(출력물).

> ⚠ **이 폴더는 `project/result/`(백데이터·입력)와 다르다.** 여기(`flight-route-advisor/result/`)는 **완료 검증 산출(출력)**이다. 백데이터에는 아무것도 쓰지 않는다([../docs/08-setup-and-dev-order.md](../docs/08-setup-and-dev-order.md) §1).

## 검증 시점

각 Stage 완료 및 전체 개발 완료 시. 코드가 [../docs/07-checklist.md](../docs/07-checklist.md)의 항목을 모두 통과한 뒤 수행한다.

## 검증 절차

1. **대조 매핑** — 아래 초기 문서의 명세와 구현을 1:1로 대조한다.
   - [../docs/03-backend-api.md](../docs/03-backend-api.md): 엔드포인트별 존재·요청/응답·데이터 출처
   - [../docs/04-frontend-migration.md](../docs/04-frontend-migration.md): 레이어별 데이터 소스·렌더
   - [../docs/05-mvp-scope.md](../docs/05-mvp-scope.md) / [../docs/07-checklist.md](../docs/07-checklist.md): 작업 항목·Stage 범위
   - [../docs/02-db-integration.md](../docs/02-db-integration.md): 최신본 규약·컬럼 매핑·읽기전용
2. **리뷰에이전트 최종 통과 확인** — `code-reviewer`/`senior-code-reviewer`로 ①기능 버그 ②논리 버그 ③예외처리 ④보안 취약점(시큐어코딩)을 최종 점검, 잔여 지적 0.
3. **시큐어코딩 체크리스트** — 입력검증·파라미터라이즈드 쿼리·경로주입 차단·비밀정보 비커밋·최소권한 role·CORS·의존성 고정·오류 비노출([../docs/06-conventions.md](../docs/06-conventions.md)).
4. **결과 등급 판정** — 각 항목을 `일치 / 부분 / 불일치·미구현`으로 표기.
5. **불일치 처리** — 불일치·미구현이 있으면 **임의로 문서나 코드를 바꾸지 않고**, 사용자에게 (a) 차이 내용, (b) 원인(설계 변경 vs 구현 누락), (c) 처리안(문서 갱신 vs 코드 보완)을 **설명하고 어떻게 할지 질문**한다.

## 산출물

- `review-<날짜>.md` — 아래 템플릿으로 작성한 대조 리뷰 리포트.
- `PROGRESS.md`(선택) — 중단/재개용 진행 메모.
