UI 설계 제안 · 의사결정자 관점

전세계 지도를 다 그리지 않는다.
지금 결정하는 루트만 보여준다.

완성본 지도는 전세계 항공로 25만+ 도형을 상시 렌더한다. 관제·운항 의사결정자에게 필요한 것은 “세계 전체”가 아니라 “이 비행을 어디로 보낼까”다. 세계 데이터는 그대로 보유하되, 기본 화면은 선택한 출·도착과 후보 루트의 FIR·항공로만 보여주는 결정 중심(cockpit) UI를 제안한다.

기준 완성본_전세계_항공로_지도.html 데이터 ODR2·ACDM·FOIS·흐름관리 (processed_*) 정체성 계승 ink/paper · orange #e8590c · blue #1d5fae

핵심 제안. 전세계 지도는 “옵션(전세계 모드)”으로 두고, 기본은 결정 포커스로 전환한다. 사용자가 OD와 후보 루트를 고르면 그 경로가 지나는 FIR·항공로·트랙만 그리고 화면을 자동으로 맞춘다(fit-bounds). 정보 과부하와 렌더 비용이 함께 줄고, 결정 근거(경유 FIR·상층풍·지연원인·흐름관리)가 한 화면에 모인다.

02

세 가지 뷰 모드

기본

① 결정 포커스

선택한 OD·후보 루트의 경유 FIR·항공로·트랙만. 도형 수 급감 → 빠르고 모바일에도 적합. 결정에 필요한 것만 남긴다.

주변

② 지역 컨텍스트

인천 FIR + 인접 FIR·관제 섹터. “옆에 무엇이 있는지” 맥락을 확인할 때. 포커스와 전세계의 중간.

옵션

③ 전세계

완성본과 동일한 전 레이어. 탐색·전체 조망용. 버리지 않고 온디맨드로 로드 — 필요할 때만 부른다.

03

기상 자료 표시 방식 — 제안

원칙: 결정 영향 순서로 계층화하고, 위험도 색(초록·주황·빨강)은 브랜드 accent(orange/blue)와 분리한다. 기본은 최소로 얹고, 상세는 토글·클릭으로. 위 목업 지도의 “기상” 칩으로 각 레이어를 켜고 끌 수 있다.

레이어표시 방식기본단계
① 경로 위 시어·상층풍선택 루트를 CAT 시어 등급으로 세그먼트 색칠(≤4/4–7/>7). 고도 변경 시 갱신 → “어느 고도·루트가 안전·빠른가”에 직결미표시2단계(04-E)
② 공항 기상 카테고리출·도착점에 VFR/MVFR/IFR/LIFR 색 뱃지, 클릭 시 METAR/TAF 팝업(실황·예보 타임라인)항상MVP
③ 경유 FIR 심각도포커스 FIR 면에 강수·위험 틴트(옅게). 전세계가 아니라 경유 FIR만 → 과부하 없음항상MVP
④ 위험기상(SIGMET/PIREP)근거 패널에 칩(TS·ICE·TURB) + 루트와 교차하면 지도에 경고 마커조건부MVP
⑤ 기상 레이더(RainViewer)토글 레이어 + 재생 바(과거 2h). 결정 포커스에선 기본 OFF, 필요 시 ON(온디맨드)토글MVP

우측 “의사결정 근거” 패널은 수치 요약을 담당 — 지도 오버레이(공간)와 패널(수치)을 역할 분리한다. MVP는 경유 FIR 체인만 채운다. 배풍/정풍·측풍·시어 등급·★추천 고도(①)·지연원인(FOIS)·흐름관리 상태는 2단계 진입 후 추가되며, 위 목업(01 콕핏)의 우측 패널·후보 루트 “정시%”·좌측 실시간 배지는 2단계 이후 완성 상태를 보여주는 목표 화면이다.

05

완성본 대비 무엇이 나아지나

완성본 (현재)
  • 전세계 25만+ 도형 상시 렌더
  • 결정 대상까지 탐색 필요
  • 단일 HTML ~15MB 임베드
  • 근거(기상·정시성·흐름)가 분산
결정 콕핏 (제안)
  • 선택 루트의 FIR·항공로만 (자동 fit)
  • OD 선택 → 후보·근거 즉시 정렬
  • API+DB 동적 로딩(참조는 캐시)
  • 근거 3-패널로 한 화면 통합
06

향후 — 실시간 의사결정 구조

비행계획서·ACDM·FOIS 전처리 데이터가 실시간 수신되면, 그 즉시 판단이 필요한 항목을 “지금 결정 큐”로 우선순위화해 콕핏 상단에 띄운다. (현 MVP는 배치, 이는 향후 단계 설계)

지금 결정 큐
14:02KAL672 흐름관리 MIT 발효 — 경로 영향보기 →
14:05AAR521 지연 위험 상승 (ACDM 예측)대안 →
14:09RKSI→RJTT 상층풍 변화 — 고도 재검토보기 →

항목 클릭 → 해당 OD·루트로 결정 포커스 자동 전환

flowchart LR
  SRC["실시간 소스\n비행계획서·ACDM·FOIS"] --> ING["실시간 수신·전처리\n(스킬 재사용)"]
  ING --> BUS["이벤트 버스"]
  BUS --> DEC["결정 엔진\n지연위험·흐름발효·경로영향"]
  BUS --> STORE["processed_* + 실시간 뷰"]
  DEC --> Q["지금 결정 큐"]
  STORE --> API["advisor API"]
  Q --> API
  API --> UI["결정 콕핏 UI"]

전처리 로직은 재사용, 수신만 배치→스트리밍. 읽기 전용·최신본 규약 유지.
