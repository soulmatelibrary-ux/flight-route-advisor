# 데이터적재 백엔드 — 개발 문서 진입점

- 문서 버전: 1.0
- 작성일: 2026-07-21

이 폴더(`data-ingestion-backend/docs/`)는 **1단계 데이터적재 백엔드를 코드 없이 문서만 읽고 바로 구현 착수**할 수 있도록 작성한 개발 문서 묶음이다. Claude 또는 개발자가 새 세션에서 이 폴더만 받아도 구현을 시작할 수 있는 것이 목표다.

## 이 프로젝트가 하는 일 (한 줄)

항공데이터(비행자료/ACDM/FOIS/흐름관리일지) 원본 파일을 웹에서 수동 업로드하면, **이 저장소의 기존 전처리 스킬(`skills/`)을 그대로 subprocess로 호출**해 전처리하고, **원본(raw)과 전처리 결과(processed)를 PostgreSQL에 적재**하는 백엔드. 로컬 Docker Postgres로 시작해 향후 Supabase로 이전한다.

## 반드시 지킬 불변 원칙 (프로젝트 `CLAUDE.md` 연장)

1. 원본 폴더(`공간데이터/`, `비행자료/`, `ACDM/`, `FOIS/`, `흐름관리일지/`)와 스킬 코드(`skills/*/scripts/*.py`)는 **수정·삭제하지 않는다**. 결과물은 백엔드의 `data/`와 DB에만 만든다.
2. 파싱/전처리 로직을 **재구현하지 않는다.** 스킬을 블랙박스 CLI로 subprocess 호출한다.
3. **표준화·저하드코딩**: 스킬별 인터페이스 차이는 `SkillDescriptor` 레지스트리로 데이터화하고, 매직값(경로/컬럼명/정규식)은 `constants.py`·`registry.py`·`DB스키마 §9`에만 둔다.
4. **완료 판정**: 실제 원본 폴더를 백엔드로 처리한 결과가 "스킬 직접 실행 기준선"과 완전히 일치해야 완료다(작업계획서 §10).

## 현재 상태
