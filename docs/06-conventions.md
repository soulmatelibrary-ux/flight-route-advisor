# 06. 이식 규약·함정 + 하드코딩 최소화 원칙

- 문서 버전: 1.1

- 관련 문서: [02-db-integration](./02-db-integration.md), [03-backend-api](./03-backend-api.md), [04-frontend-migration](./04-frontend-migration.md), [07-checklist](./07-checklist.md)
- 원본 근거: `비행경로추천서비스_이식패키지/문서/05_트러블슈팅.md`, `08_임베드데이터_스키마.md`

이 문서는 구현 시 **반드시 지켜야 할 규약**과 **재현 실패의 단골 함정**을 모은다. (원본 `05`는 "재현 실패의 90%가 규약 위반"이라 명시)

## 1. 하드코딩 최소화 (설정 주도) — 최우선 원칙

**모든 환경 의존값·매직 넘버·URL·자격증명은 코드에 박지 않고 설정으로 뺀다.**

### 1.1 백엔드
| 값 | 위치 | 금지 예 |
|---|---|---|
| `DATABASE_URL`, DB 자격증명 | `.env` → `config.py` | 코드에 접속문자열 리터럴 |
| SSL/pool/pooler 분기 | 환경변수 | `sslmode=require` 상수 박기 |
| 참조 아티팩트 경로 | 설정 | 절대경로 하드코딩 |
| 캐시 TTL, 페이지 크기, bbox 기본값 | 설정 | 라우터에 `max_age=86400` 직접 |
| 논리↔물리 컬럼명 | `db/column_map.py` 한 곳 | 쿼리마다 물리명 산재 |
| 표시 임계(픽스 800, 라벨 줌 등) | 설정 | 쿼리/필터에 리터럴 |

- 자격증명은 **절대 커밋 금지**(`.env`는 `.gitignore`). `.env.example`만 커밋.
- 조직 정책상 민감정보(접속정보·키)는 문서·코드에 직접 쓰지 않는다.

### 1.2 프론트
- API 베이스 URL·기상 프록시 URL·타일 URL·외부 API URL → `config.js`/`config.json`([04](./04-frontend-migration.md) §5).
- 디자인 토큰(색·폰트)·지도 초기값(center/zoom/bounds)·표시 상한·라벨 줌 임계 → config.
- **금지**: 컴포넌트에 `http://localhost:3000`, `#e8590c`, `800`, `[30,127]` 같은 리터럴 직접 사용.

### 1.3 단일 진실원(Single Source of Truth)
- 외부 API URL·프록시 화이트리스트는 기상서버(`07`)와 프론트가 **같은 목록**을 참조(불일치 = CORS 폴백 실패).
- 좌표·시간 규약(아래 2·3장)은 백엔드·배치·프론트가 동일하게 적용.

## 2. 좌표 규약

1. **순서는 항상 `[lat, lon]`** (위도, 경도). GeoJSON(`[lon, lat]`)과 반대 — 변환 시 반드시 뒤집는다.
2. **flat 폴리곤**: `[lat0,lon0,lat1,lon1,...]` 1차원. API↔앱 변환 시 형태 유지.
3. **정밀도**: 좌표 소수 3~4자리 반올림(용량 절감). 정밀도를 올리면 데이터량 급증(원본 15MB→).
4. **FIR 폴리곤 경도 연속화**: 날짜변경선(±180) 넘을 때 ±360 보정된 상태 유지. bbox·클릭 판정이 여기 의존.
5. **ZBPE(베이징) 오류 폴리곤 제거**: 중국 전역을 덮는 잘못된 폴리곤(ZB00008) 제외 상태 유지. 라벨은 bbox 중심.

## 3. 시간 규약

1. **모든 운항 시각은 KST.** 원자료가 KST이며 **+9 재변환 절대 금지**(원본 `05` 1순위 함정, 전처리 `작업계획서`도 동일). 지연 = ATD−EOBT(15~600분만 카운트).
2. 기상(AWC)의 `obsTime` 등 UTC epoch는 표시 시에만 `UTC+9`로 KST 병기(원본 `07` `kst()`), 저장·집계값과 혼동 금지.
3. TAF 타임라인 카테고리 임계·TEET "HH:MM" 파싱 등은 원본 `04-G` 규칙 그대로.

## 4. 데이터 구조 규약

1. **인덱스 의존 금지(신규 규약).** 원본 앱은 `route[4]`, `a[5]`처럼 배열 위치로 읽었다. 동적 전환 후 **API 경계에서는 키 기반**을 쓰고, 배열 접근은 프론트 어댑터 내부로만 국한([04](./04-frontend-migration.md) §4). 새 필드는 끝에 추가.
2. **문자열 vs 숫자.** 고도(elevFt, upper/lower)는 문자열, 좌표·편수·분은 숫자. 타입 혼용 금지(원본 `08`).
3. **ODR2 옵션 인덱스 고정.** 배치 집계가 만드는 옵션 배열의 인덱스 의미(0:n … 8:짝홀)는 원본 `08` ODR2와 동일하게 유지.
4. **동명 픽스 3단계 해석.** FULL_ROUTE 좌표해석은 원본 `04-B`(탐욕→재선택→스파이크 제거) 그대로 — 백엔드 배치로 이동해도 로직 동일. 검증 지표: 스파이크 0건.

## 5. DB 소비 규약 (append-only)

1. **최신본 규약 필수.** `processed_*`는 append-only이므로 항상 최신 SUCCESS run/기간으로 필터([02](./02-db-integration.md) §3). 단순 `SELECT *` 금지.
2. **읽기 전용.** 이 서비스는 DDL/INSERT/UPDATE 없음. 집계 산출물도 전처리 DB에 쓰지 않는다.
3. **검증 경고 전달.** `validation_summary`의 제외 건수를 API 메타로 노출(데이터 신뢰도).
4. **출처 기록.** 응답에 조회한 `run_id`·데이터 기간 포함(감사성).

## 6. 기상·외부 API 규약

1. **프록시 폴백 체인 유지**: 직접 → `localhost:3000/proxy` → 공개 프록시, 성공 경로 기억(원본 `03`/`05`). URL은 config에서.
2. **화이트리스트 일치**: 프록시 허용 호스트(AWC·RainViewer·Open-Meteo·ADS-B 3종·adsbdb)를 프론트 config와 기상서버가 동일하게.
3. **키 불필요 유지**: 외부 API는 무료·키 불필요. 새 유료 키를 도입하지 않는다(도입 시 자격증명은 설정·비커밋).
4. RainViewer maxNativeZoom 6, dBZ=v/2−32, ADS-B 반경 250NM 등 원본 `01`/`04-G` 파라미터는 config 상수로.

## 7. 검증 규약 (브라우저 없이 회귀 방지 — 원본 계승)
- JS: `node --check` 문법 검증, DOM/Leaflet 스텁 하네스로 클릭 흐름 시뮬레이션(그려진 폴리라인/마커 수).
- 참조 데이터 건수 대조: FIR 247 · 항공로 89,555 · 픽스 58,812 · 공항 10,030 · OD 1,487 · 경로 3,083(2026-01 실계획 84,490편 기반, 원본 `06_재현가이드`·`08_임베드데이터_스키마` §"규모").
- 백엔드: 쿼리 결과가 "최신 run만" 담는지, 어댑터 왕복(키↔배열)이 무손실인지 단위테스트.
- 대용량 CSV/DB·완성본 HTML(15MB)은 직접 열지 말고 짧은 pandas/SQL/스크립트로 필요한 부분만 확인(전처리 `CLAUDE.md` 규칙 계승). 완성본 HTML은 회귀 비교용 **정답지**이며 `.gitignore` 대상, `Read`로 통째 열지 않는다.

## 8. 시큐어코딩 규약 (필수)

모든 구현은 아래를 지킨다. 리뷰에이전트가 이 항목을 점검한다([07](./07-checklist.md) 공통 게이트).

| 영역 | 규칙 |
|---|---|
| 입력 검증 | 모든 외부 입력(쿼리 파라미터·업로드·API 바디)의 타입·범위·화이트리스트 검증. ICAO/날짜/좌표 형식 강제. 잘못된 입력은 `400` |
| SQL 인젝션 | 문자열 조립 쿼리 금지. **SQLAlchemy Core 바인드 파라미터**만 사용. 테이블/컬럼명은 `column_map` 화이트리스트에서만 |
| 경로 주입 | 업로드(Stage 0)·아티팩트 경로에 `../` 차단, 저장 파일명은 서버 생성명(uuid), 사용자 파일명은 메타로만 |
| 파일 업로드 | 확장자(`.csv`,`.xlsx`)·용량 화이트리스트, 콘텐츠 타입 확인 |
| 비밀정보 | DB 접속·키를 코드/문서/로그에 노출 금지. `.env`만, 커밋 금지 |
| 최소 권한 | advisor는 **읽기 전용 DB role**. DDL/쓰기 권한 부여 금지 |
| CORS | 허용 오리진 화이트리스트(와일드카드 `*` 지양, 프록시 화이트리스트와 정합) |
| **요청량 제한** | 인증 없는 읽기 전용 API라도 rate limiting 적용(리버스 프록시 또는 FastAPI 미들웨어). curl 등 직접 호출로 ODR2/집계 엔드포인트에 과부하 방지. **조직 정책(API 자동화 시 요청량 제한·로그 기록) 필수.** |
| 업로드 폭탄 | XLSX(zip) 압축 해제 후 크기·시트 수 상한 검증(원본 크기 제한만으로 불충분, decompression bomb 방지) |
| 의존성 | 버전 고정(pin). 알려진 취약 버전 회피, 주기적 점검 |
| 오류 처리 | 예외를 삼키지 말고 로깅하되, 응답에는 스택/쿼리/내부 경로 등 내부정보 비노출 |
| 로깅 | 자격증명·개인정보·전체 비행계획 원문을 로그에 남기지 않음 |

> 조직 정책: 생성 코드는 리뷰 없이 배포 금지, 보안 취약점·라이선스·성능 이슈를 반드시 검토. 민감정보 입력·하드코딩 금지.
